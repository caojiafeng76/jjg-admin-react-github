# 刀具资料复制新增设计

## 目标

在 `/tooling-data` 页面支持选中一条刀具资料后复制新增。点击后打开现有新增弹窗，并自动填充源记录的全部表单字段；用户可修改字段，点击“新增”后创建新记录。

## 交互

- 复制新增要求恰好选中一条记录。
- 未选中一条记录时提示“请选择一条数据进行复制新增”。
- 复制新增弹窗标题为“复制新增刀具资料”。
- 刀具编号也原样复制，用户可在提交前修改；后端现有唯一性校验继续生效。
- 提交成功后沿用现有新增流程：提示成功、关闭弹窗、重置表单并清空选择。

## 实现边界

- 复用 `ToolingDataPage` 现有新增弹窗、`ToolingDataForm` 和 `useCreateToolingData`，不新增服务接口或数据库字段。
- 新增一个复制新增状态，用于区分普通新增与复制新增标题；复制模式仍然是创建模式，不能进入更新分支。
- 保持现有权限、查看者保护、表单校验、缓存失效和错误提示逻辑不变。

## 测试与验证

- 为页面交互增加测试：未选中/选中多条时提示；选中一条后以创建模式打开并填充字段；提交时调用创建 mutation 而非更新 mutation。
- 运行相关 Vitest 测试、`bun lint`、`bun run build`，并对变更的 antd 组件代码运行 `bunx antd lint <changed-path> --format json`。
