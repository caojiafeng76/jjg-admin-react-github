# 生产工单开发计划

## 一、数据库设计

### 1.1 生产工单表 (production_orders)

| 字段名                | 类型          | 说明           | 约束                 |
| --------------------- | ------------- | -------------- | -------------------- |
| id                    | UUID          | 主键           | 自动生成             |
| order_date            | DATE          | 日期           | 默认为前一天         |
| employee_id           | UUID          | 操作人         | 关联 employees 表    |
| work_hours            | NUMERIC(6,2)  | 出勤工时(小时) | 必须 > 0             |
| total_qualified_hours | NUMERIC(10,2) | 合格工时(小时) | 自动计算             |
| efficiency            | NUMERIC(5,4)  | 工时效率       | 自动计算(百分比)     |
| status                | TEXT          | 状态           | 进行中/已完成/已取消 |
| remark                | TEXT          | 备注           | 可空                 |
| created_at            | TIMESTAMPTZ   | 创建时间       | 默认 now()           |
| updated_at            | TIMESTAMPTZ   | 更新时间       | 自动更新             |

### 1.2 工序明细表 (production_order_items)

| 字段名                      | 类型          | 说明           | 约束                            |
| --------------------------- | ------------- | -------------- | ------------------------------- |
| id                          | UUID          | 主键           | 自动生成                        |
| order_id                    | UUID          | 生产工单ID     | 关联 production_orders          |
| project_no                  | TEXT          | 项目号         | 必填，关联 sales_orders         |
| product_model               | TEXT          | 型号           | 自动从 sales_orders 填充        |
| length_mm                   | NUMERIC(10,2) | 长度(mm)       | 自动从 sales_orders 填充        |
| customer_model              | TEXT          | 客户型号       | 自动从 sales_orders 填充        |
| operation                   | TEXT          | 工序           | 必填，从 process_standards 获取 |
| standard_seconds            | NUMERIC(10,2) | 标准工时(秒)   | 必填，自动获取                  |
| incoming_qualified_quantity | INTEGER       | 来料合格数     | 必填                            |
| qualified_quantity          | INTEGER       | 合格数量       | 默认为 0                        |
| qualified_hours             | NUMERIC(10,2) | 合格工时(小时) | 自动计算                        |
| defect_quantity_1           | INTEGER       | 不良数量-原因1 | 默认为 0                        |
| defect_reason_1             | TEXT          | 不良原因1      | 加工/原料/其他                  |
| defect_quantity_2           | INTEGER       | 不良数量-原因2 | 默认为 0                        |
| defect_reason_2             | TEXT          | 不良原因2      | 加工/原料/其他                  |
| defect_hours                | NUMERIC(10,2) | 减分工时(小时) | 自动计算                        |
| bonus_seconds               | INTEGER       | 加分项(秒)     | 默认为 0                        |
| created_at                  | TIMESTAMPTZ   | 创建时间       | 默认 now()                      |
| updated_at                  | TIMESTAMPTZ   | 更新时间       | 自动更新                        |

### 1.3 触发器

- `trigger_update_production_order_totals`: 当工序明细增删改时，自动更新生产工单的 total_qualified_hours 和 efficiency

## 二、计算逻辑

### 2.1 合格工时 (total_qualified_hours)

```
合格工时 = (所有明细的合格工时 + 加分项 - 减分项) / 3600
```

- 合格工时 = 合格数量 × 标准工时
- 减分项 = 不良原因="加工" 的不良数量 × 2 × 标准工时

### 2.2 工时效率 (efficiency)

```
工时效率 = 合格工时 / 出勤工时
```

显示为百分比，保留 2 位小数。

### 5.2 组件功能

#### ProductionOrderList

- 工单列表表格
- 列：日期、操作人、出勤工时、合格工时、工时效率、状态、操作
- 支持分页、筛选（日期范围、操作人、状态）
- 操作：查看、编辑、删除

#### ProductionOrderForm / ProductionOrderDetail

- 日期选择器（默认前一天）
- 操作人下拉（关联 employees 表）
- 出勤工时输入（数字，单位：小时）
- 状态选择
- 备注输入
- 工序明细列表（可展开）

#### ProductionOrderItemTable

- 工序明细表格
- 列：项目号、型号、工序、标准工时、合格数量、合格工时、不良原因1/数量、不良原因2/数量、加分项、减分项
- 操作：编辑、删除

#### ProductionOrderItemForm（模态框）

- 项目号选择（关联 sales_orders，自动带出型号、长度、客户型号）
- 工序选择（根据型号从 process_standards 过滤）
- 标准工时（自动填充）
- 合格数量输入
- 不良原因1 + 不良数量1
- 不良原因2 + 不良数量2
- 加分项输入（秒）

## 六、交互逻辑

### 6.1 选择项目号后

1. 根据 project_no 查询 sales_orders
2. 自动填充 product_model、length_mm、customer_model
3. 根据 product_model 查询 process_standards 获取可用工序列表

### 6.2 选择工序后

1. 从 process_standards 获取该型号+工序的 standard_seconds
2. 自动填充标准工时

### 6.3 计算字段

- qualified_hours: 实时计算（前端显示）
- defect_hours: 实时计算（前端显示）
- total_qualified_hours: 触发器自动计算（数据库）
- efficiency: 触发器自动计算（数据库）
